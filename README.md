# 📘 KeyKeyStudy — 교과서 기반 AI 3단계 성장형 타자 학습 서비스

> **KeyKeyStudy**는 _Keyword를 Keyboard로 Study한다_ 는 의미를 담은 이름입니다.
> 교과서의 핵심 개념(Keyword)을 키보드(Keyboard)로 직접 타이핑하며 공부(Study)하는 완전 학습 플랫폼입니다.

## 💡 개발 배경

### 교과서를 끝까지 읽는 학생은 드뭅니다

많은 학생들이 수업 중 교과서 본문을 **눈으로만 훑고** 넘깁니다. 수업이 끝나도 핵심 개념이 머릿속에 남지 않는 이유입니다. 교과서 전체 내용을 스스로 정독하는 학생은 극소수에 불과하다는 현장 경험이 이 프로젝트의 출발점입니다.

### 손을 움직이며 공부하면 학습 효과가 높아집니다

학습 과학(Learning Science) 연구에 따르면, **손을 움직이는 능동적 학습(Active Recall)** 은 수동적 읽기에 비해 장기 기억 형성에 훨씬 효과적입니다. 직접 손가락으로 타이핑하며 교과서 문장을 따라 치는 행위는 반복 노출과 운동 기억을 동시에 자극하여 학습 내용을 더 깊이 각인시킵니다.

### 학생들의 타자 실력은 생각보다 낮습니다

디지털 기기 사용이 보편화되었지만, 많은 학생들이 정확한 키보드 타이핑에 익숙하지 않습니다. 타자 연습은 단순한 기술 향상을 넘어 **정보 처리 능력과 집중력** 향상에도 직접적인 영향을 줍니다. 교과 학습과 타자 연습을 하나로 묶으면 두 마리 토끼를 잡을 수 있습니다.

### 이 세 가지 문제를 동시에 해결합니다

본 서비스는 위의 세 가지 한계를 한 번에 극복하기 위해 설계되었습니다.

- **교과서 본문 전체**를 타자로 치게 함으로써 완전 정독을 유도
- **핵심 어휘 타자 → 퀴즈**로 이어지는 3단계 구조로 깊은 이해 달성
- **실시간 수업 순위표** 경쟁을 통해 학습 동기를 자연스럽게 극대화

## ✨ 주요 기능

### 1. 🤖 교과서 PDF 업로드 → AI 자동 학습 콘텐츠 생성

교사가 교과서 PDF를 업로드하고 페이지 범위만 입력하면, **Google Gemini AI**가 자동으로 분석하여 3단계 학습에 필요한 모든 콘텐츠를 생성합니다. 교사가 직접 문제를 만드는 수고가 완전히 사라집니다.

| AI가 자동 생성하는 것 | 설명 |
|---|---|
| **1단계용 본문 문장 목록** | 지정 페이지의 교과서 본문을 타자 연습용 문장 배열로 변환 |
| **2단계용 핵심 어휘 목록** | 본문에서 핵심 단어와 그 뜻·설명 문장을 자동 추출 |
| **3단계용 AI 퀴즈** | 빈칸 채우기형 + 단어 맞추기형 4지선다 객관식 퀴즈 자동 생성 |

- AI 결과물은 **임시저장(draft) 상태**로 먼저 저장됩니다.
- 교사가 미리보기에서 문장·퀴즈를 직접 수정할 수 있습니다.
- 교사가 최종 확인 후 **배포 버튼**을 누르면 학생 화면에 공개됩니다.

---

### 2. ⌨️ 타자 기반 3단계 학습 프로토콜

세 단계가 순서대로 잠금 해제되며, 앞 단계를 완료해야 다음 단계로 진입할 수 있습니다. **각 단계는 100점 만점**이며, 단계별 순위와 3단계를 합산한 종합 순위를 모두 확인할 수 있습니다.

```
1단계 완료 → 2단계 잠금 해제 → 2단계 완료 → 3단계 잠금 해제
```

| 단계 | 만점 | 순위표 |
|---|---|---|
| 1단계 본문 타자 | **100점** | 1단계 단계별 순위 |
| 2단계 핵심 어휘 타자 | **100점** | 2단계 단계별 순위 |
| 3단계 어휘 퀴즈 | **100점** | 3단계 단계별 순위 |
| **종합** | **300점** | **1~3단계 합산 종합 순위** |

#### 📖 1단계: 본문 타자 연습 (전체 문맥 파악)

- 교과서 본문 전체를 한 줄씩 따라 치며 완전 정독을 유도합니다.
- 타자 속도(WPM)와 정확도(%)를 실시간 측정하여 **100점 만점**으로 점수를 산출합니다.
- 완료 즉시 **1단계 순위표**에 반영되며, 같은 반 친구들과 실시간으로 경쟁합니다.

#### 📝 2단계: 핵심 어휘 타자 연습 (개념 집중 인지)

- 1단계 완료 후 잠금이 해제됩니다.
- AI가 추출한 핵심 단어의 **뜻과 설명 문장**만 골라서 타자 연습합니다.
- 1단계와 동일하게 타자 속도(WPM)와 정확도(%)를 실시간 측정하여 **100점 만점**으로 점수를 산출합니다.
- 완료 즉시 **2단계 순위표**에 반영됩니다.

#### 🏆 3단계: 핵심 어휘 퀴즈 (완전 학습 검증)

- 2단계 완료 후 잠금이 해제됩니다.
- 2단계에서 학습한 어휘를 기반으로 2가지 유형의 퀴즈가 출제됩니다.

| 유형 | 설명 |
|---|---|
| **빈칸 채우기형** | 핵심 문장에서 중요 어휘 자리에 빈칸을 뚫고, 4지선다 중 정답 선택 |
| **단어 맞추기형** | 제시된 단어 설명을 읽고, 해당하는 단어를 4지선다 중 선택 |

- 정답 수만을 기준으로 **100점 만점**으로 최종 점수가 산출됩니다.
- 완료 즉시 **3단계 순위표**와 **1~3단계 종합 순위표** 양쪽에 동시에 반영됩니다.
- 퀴즈를 푸는 도중 이해가 안 되는 개념은 화면 내 **AI 챗봇**에게 바로 질문할 수 있습니다. 챗봇은 해당 차시 학습 내용을 기반으로 쉬운 언어로 개념을 추가 설명해 줍니다.

---

### 3. 🏫 수업 및 학생 관리

- **수업 생성 및 코드 발급:** 교사가 수업을 개설하면 고유한 6자리 수업 코드가 생성됩니다. 학생은 이 코드로 수업에 참여합니다.
- **직접 초대:** 교사는 교사·학생을 직접 초대할 수도 있습니다.
- **권한 체계:** 교사(teacher) / 학생(student) 2단계 권한으로 기능을 분리합니다. 교사는 수업내 학생 정보(이름 등)를 직접 수정할 수 있습니다.
- **단계별 · 종합 순위표:** 각 단계(1·2·3단계) 순위표와 전 단계 합산 **종합 순위표**를 별도로 운영합니다. 학생은 특정 단계에서 강점을 보이거나 종합 1위를 노리는 등 다양한 방식으로 경쟁할 수 있습니다.
- **대시보드 통계:** 수업별·차시별 학습 현황, 완료율, 단계별 점수 분포를 한눈에 확인합니다.
- **순위표 단위 초기화:** 특정 학생의 성적 고착화로 인한 의욕 저하를 막기 위해, 순위표는 차시/단원 단위로 분리·초기화되어 운영됩니다.

## 👨‍🏫 사용 방법 — 교사

### 0단계. 사전 준비 (최초 1회)

1. **구글 계정으로 로그인**
   서비스 접속 후 `Google로 로그인` 버튼을 클릭합니다. 구글 계정이 있다면 누구나 로그인 시도가 가능합니다.

2. **교사 인증코드 입력**
   최초 로그인 시 **교사 인증코드 입력 화면**이 자동으로 표시됩니다. 관리자에게 발급받은 인증코드를 입력하면 교사 권한이 부여됩니다. 인증코드가 없으면 교사 기능을 사용할 수 없습니다.

---

### 1단계. 수업 개설

1. 교사 대시보드 접속 후 **[+ 새 수업 만들기]** 버튼 클릭
2. 수업명 입력 (예: `3학년 2반 과학`)
3. 수업이 생성되면 **6자리 수업 코드**가 자동 발급됩니다.
4. 이 코드를 학생들에게 공유합니다. (칠판에 적거나 구두로 전달)
5. 또는 **[교사/학생 직접 초대]** 기능을 통해 초대 링크를 생성하여 공유할 수도 있습니다.

---

### 2단계. 교과서 PDF 업로드 및 AI 콘텐츠 생성

1. 해당 수업을 선택하고 **[새 차시 콘텐츠 만들기]** 클릭
2. 다음 정보를 입력합니다:

   | 입력 항목 | 예시 |
   |---|---|
   | 과목명 | `과학` |
   | 단원명 | `3. 자석의 이용` |
   | 차시 제목 | `1차시. 자석이 붙는 물체` |
   | 학습할 페이지 범위 | `48 ~ 51` |

3. 교과서 PDF 파일을 업로드합니다.
4. **[AI 콘텐츠 생성 요청]** 버튼을 클릭합니다. (최대 30초 소요)
5. AI가 1·2·3단계 전체 콘텐츠를 자동으로 생성합니다.

---

### 3단계. AI 결과물 검토 및 수정

1. 생성된 콘텐츠가 **미리보기 화면**에 표시됩니다.
2. 1단계 본문 문장, 2단계 핵심 어휘, 3단계 퀴즈 문항을 순서대로 검토합니다.
3. 수정이 필요한 항목은 **직접 편집**할 수 있습니다.
4. 전체적으로 결과가 부적절하다면 **[AI에게 재생성 요청]** 버튼으로 다시 생성합니다.

---

### 4단계. 학생에게 배포

1. 검토가 완료되면 **[배포하기]** 버튼을 클릭합니다.
2. 즉시 해당 수업 학생들의 화면에 차시 콘텐츠가 공개됩니다.
3. 교사 대시보드에서 실시간으로 학생들의 학습 진행 현황을 모니터링할 수 있습니다.

---

### 5단계. 결과 확인

- **수업 대시보드**에서 차시별 완료 학생 수, 평균 타수, 정확도, 점수를 확인합니다.
- **순위표**에서 개인별 점수를 조회하고 학생 피드백에 활용합니다.

---

## 🧑‍🎓 사용 방법 — 학생

### 0단계. 사전 준비 (최초 1회)

1. **구글 계정으로 로그인**
   서비스 접속 후 `Google로 로그인` 버튼을 클릭합니다.

2. **수업 참여 — 아래 두 가지 방법 중 하나를 사용합니다**

   | 방법 | 설명 |
   |---|---|
   | **수업 코드 입력** | 선생님이 알려준 **6자리 수업 코드**를 입력하면 해당 수업에 자동 참여됩니다 |
   | **선생님 초대 받기** | 선생님께 수업을 초대해 달라고 말씀드립니다 |

---

### 1단계. 학습할 차시 선택

1. 학생 메인 화면에 선생님이 배포한 **차시 목록**이 표시됩니다.
2. 학습할 차시를 클릭하면 3단계 학습 화면으로 진입합니다.

---

### 2단계. 1단계 — 본문 타자 연습

1. 화면에 교과서 본문 문장이 **한 줄씩** 표시됩니다.
2. 키보드로 화면에 표시된 문장을 **그대로 따라 입력**합니다.
3. 입력한 내용은 실시간으로 정오답 색상(초록/빨강)으로 표시됩니다.
4. 모든 문장을 완료하면 타자 속도와 정확도를 기반으로 **100점 만점** 점수가 자동 계산됩니다.
5. 점수가 즉시 **1단계 순위표**에 반영됩니다.

> ⚠️ **주의:** 학교 와이파이가 불안정해도 걱정하지 마세요. 진행 상황이 내 기기에 자동으로 임시 저장되며, 연결이 복구되면 자동으로 서버에 동기화됩니다.

---

### 3단계. 2단계 — 핵심 어휘 타자 연습

1. 1단계 완료 후 **2단계 잠금이 해제**됩니다.
2. 이번 차시의 핵심 단어와 그 뜻·설명 문장이 순서대로 제시됩니다.
3. 화면에 표시된 **핵심 어휘 설명 문장을 그대로 입력**합니다.
4. 손으로 직접 치면서 중요한 개념을 확실하게 익힙니다.
5. 모든 어휘 문장을 완료하면 1단계와 동일하게 타자 속도와 정확도 기반 **100점 만점** 점수가 산출되고, **2단계 순위표**에 반영됩니다.

---

### 4단계. 3단계 — 핵심 어휘 퀴즈

1. 2단계 완료 후 **3단계 잠금이 해제**됩니다.
2. AI가 출제한 퀴즈 문항이 화면에 표시됩니다.

   - **빈칸 채우기형:** 문장에서 빠진 핵심 단어를 4개 보기 중 선택
   - **단어 맞추기형:** 제시된 단어 설명을 읽고 해당 단어를 4개 보기 중 선택

3. 문제를 푸는 도중 모르는 개념이 있으면 화면 내 **🤖 AI 챗봇**에게 바로 질문할 수 있습니다.
   - 챗봇은 해당 차시 교과서 내용을 기반으로 쉬운 언어로 개념을 추가 설명합니다.
   - 단순 정답을 알려주지 않고, 이해를 도울 수 있도록 힌트와 설명을 제공합니다.
4. 퀴즈를 완료하면 정답 수만을 기준으로 **100점 만점** 최종 점수가 산출됩니다.
5. 점수가 **3단계 순위표**와 **1~3단계 종합 순위표** 양쪽에 즉시 반영됩니다.
6. 각 문항의 해설을 확인하며 틀린 부분을 복습합니다.

## 🏗 시스템 아키텍처

```
[ 교사 / 학생 브라우저 ]
        │
        ▼
[ Next.js 14 (Vercel 배포) ]
  ┌─────────────────────────┐
  │  페이지 라우팅 / API Routes  │
  └────────────┬────────────┘
               │
    ┌──────────┴──────────┐
    │                     │
    ▼                     ▼
[ Firebase ]         [ Google Gemini API ]
  ├─ Authentication        PDF 분석 및 콘텐츠 생성
  ├─ Firestore             (최대 30초 응답 목표)
  │   └─ app_store (단일 컬렉션)
  │       ├─ whitelist 문서
  │       ├─ class 문서
  │       ├─ content 문서
  │       └─ score 문서
  └─ Cloud Storage
      └─ 교과서 PDF 파일
```

### 데이터 흐름

1. 교사가 PDF 업로드 → Cloud Storage 저장
2. API Route에서 PDF 텍스트 추출 → Gemini API 전송
3. Gemini가 1·2·3단계 JSON 데이터 팩 반환
4. Firestore `content` 문서에 `status: draft`로 저장
5. 교사 검토 후 배포 → `status: published`로 전환
6. 학생이 학습 완료 → Firestore `score` 문서에 실시간 저장
7. 모든 학생의 점수가 실시간 동기화 → 순위표 즉시 업데이트
